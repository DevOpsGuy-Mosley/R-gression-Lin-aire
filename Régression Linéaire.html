Régression Linéaire

Fondamentaux du Machine Learning Supervisé

1. Contexte : IA, Machine Learning et Apprentissage Supervisé

Pour bien comprendre la régression linéaire, il faut la situer.

Intelligence Artificielle (IA) :

Le domaine global. L'objectif est de créer des machines qui simulent des capacités cognitives humaines (résolution de problèmes, apprentissage, etc.).

Machine Learning (ML) :

Une sous-catégorie de l'IA. Il s'agit de donner aux machines la capacité d'apprendre à partir de données sans être explicitement programmées pour chaque tâche.

Apprentissage Supervisé :

Une sous-catégorie du ML. L'algorithme apprend à partir d'un ensemble de données "étiquetées", c'est-à-dire où chaque entrée a une sortie correspondante. La machine apprend à prédire la sortie pour de nouvelles entrées. C'est ici que se trouve la régression linéaire.

2. Qu'est-ce que la Régression Linéaire ?

La régression linéaire est un des algorithmes d'apprentissage supervisé les plus simples et les plus fondamentaux. Son objectif est de modéliser la relation entre une variable indépendante (ou plusieurs) et une variable dépendante en ajustant une ligne droite (ou un hyperplan dans le cas multidimensionnel) à ces données.

Variable indépendante (x) :

La(les) variable(s) que nous utilisons pour faire la prédiction. On l'appelle aussi "caractéristique" ou "feature".

Variable dépendante (y) :

La variable que nous essayons de prédire. On l'appelle aussi "cible" ou "target".

Formule Mathématique de Base

La forme la plus simple est la régression linéaire simple (une seule variable indépendante) :

y = ax + b
  • y : La valeur prédite.
  • x : La valeur de la variable indépendante.
  • a : Le "coefficient" ou la "pente" de la ligne. Il représente le changement de y pour chaque unité de changement dans x.
  • b : L'"ordonnée à l'origine" ou "biais". C'est la valeur de y lorsque x est égal à 0.

L'objectif de l'algorithme est de trouver les meilleures valeurs pour a et b pour que la ligne soit la plus proche possible de tous les points de données.

3. Comment ça Marche ? - Le Cœur de l'Apprentissage

L'apprentissage de la régression linéaire consiste à minimiser l'erreur entre les valeurs prédites par notre modèle (y_prédit) et les valeurs réelles (y_réel).

Fonction de Coût (Cost Function)

Pour mesurer cette erreur, on utilise une fonction de coût. La plus courante pour la régression linéaire est l'Erreur Quadratique Moyenne (Mean Squared Error - MSE).

MSE = (1/n) * Σ(y_i - ŷ_i)²
  • n : Le nombre de points de données.
  • y_i : La valeur réelle du ième point.
  • ŷ_i : La valeur prédite par le modèle pour le ième point.

L'objectif de l'algorithme est de trouver les paramètres (a et b) qui minimisent ce MSE.

Méthode pour minimiser le coût

Deux méthodes principales :

Méthode des moindres carrés ordinaires (OLS - Ordinary Least Squares) :

Une méthode analytique qui permet de trouver directement les valeurs optimales de a et b en résolvant une équation. C'est très efficace pour les petits jeux de données.

Descente de gradient (Gradient Descent) :

Une méthode itérative. On part de valeurs aléatoires pour a et b et on les ajuste petit à petit dans la direction qui réduit le plus la fonction de coût, jusqu'à atteindre un minimum local (ou global). C'est la méthode de choix pour les grands jeux de données.

4. Exemples Concrets pour un Développeur

Voici des scénarios où vous pourriez utiliser la régression linéaire.

Exemple 1 : Prédiction du prix d'une maison

  • Problème : Prédire le prix de vente d'une maison en fonction de sa surface.
  • Variables :
    • x (indépendante) : Surface de la maison en m².
    • y (dépendante) : Prix de la maison en euros.
  • Modèle : prix = a * surface + b
import numpy as np
from sklearn.linear_model import LinearRegression

# Données d'exemple : (surface, prix)
X = np.array([50, 75, 100, 120, 150]).reshape(-1, 1) # Surface en m²
y = np.array([150000, 220000, 290000, 350000, 450000]) # Prix en €

# Création et entraînement du modèle
model = LinearRegression()
model.fit(X, y)

# Afficher les coefficients (a et b)
print(f"Coefficient (a) : {model.coef_[0]:.2f}")
print(f"Ordonnée à l'origine (b) : {model.intercept_:.2f}")

# Utilisation du modèle pour une prédiction
surface_nouvelle = 110 # Une maison de 110m²
prix_predit = model.predict([[surface_nouvelle]])
print(f"Prix prédit pour une maison de {surface_nouvelle}m² : {prix_predit[0]:.2f} €")

Exemple 2 : Régression Linéaire Multiple

La régression linéaire peut gérer plusieurs variables indépendantes. La formule devient :

y = a₁x₁ + a₂x₂ + ... + aₙxₙ + b
  • Problème : Prédire les ventes d'un produit en fonction du budget publicitaire sur la télévision et la radio.
  • Variables :
    • x₁ : Budget TV
    • x₂ : Budget Radio
    • y : Ventes
  • Modèle : ventes = a₁ * budget_tv + a₂ * budget_radio + b

L'interprétation devient : a₁ représente l'impact des dépenses TV sur les ventes (en maintenant la radio constante), et a₂ l'impact des dépenses radio (en maintenant la TV constante).

5. Veille Technologique pour un Développeur

Votre rôle ne se limite pas à écrire le code. Voici ce qu'un développeur doit savoir en plus :

Limites de la Régression Linéaire :

  • Hypothèse de Linéarité : Elle suppose que la relation entre les variables est linéaire. Si ce n'est pas le cas, le modèle sera mauvais.
  • Sensibilité aux Outliers : Les valeurs extrêmes (outliers) peuvent fortement biaiser la ligne de régression.
  • Pas pour les données de classification : C'est un modèle de régression (prédiction d'une valeur continue), pas de classification (prédiction d'une catégorie).

Quand l'utiliser ?

  • Comme un premier modèle "baseline" simple et rapide à implémenter.
  • Pour des problèmes où on pense qu'il y a une relation linéaire (ex: taille vs poids, prix vs surface).
  • Pour l'interprétabilité : les coefficients a et b sont faciles à comprendre.

Alternatives et évolutions :

  • Régression polynomiale : Pour modéliser des relations non linéaires en transformant les données d'entrée.
  • Régression logistique : Pour les problèmes de classification. C'est l'étape suivante logique après la régression linéaire.
  • Régression Ridge et Lasso : Des versions améliorées de la régression linéaire qui aident à gérer la sur-apprentissage (overfitting) et la multicolinéarité.

Outils et Librairies à connaître (en tant que dev) :

  • scikit-learn : La bibliothèque incontournable.

  • matplotlib/seaborn : Pour la visualisation.

Autres langages :

  • C++/Java : Pour les implémentations en production.

Conclusion

La régression linéaire est la porte d'entrée de l'apprentissage supervisé. Pour un développeur, la maîtriser ne signifie pas seulement connaître sa formule, mais comprendre ses hypothèses, ses forces, ses faiblesses, et savoir l'implémenter avec les outils adéquats. Elle sert de fondation solide pour aborder des modèles plus complexes et plus puissants.
